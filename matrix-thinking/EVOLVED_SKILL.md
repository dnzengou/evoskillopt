# Matrix Thinking v2.0

## Core Premise

The Matrix is a compressed masterclass in systems, perception, and leverage. Central question: **Who built this simulation, and what are they optimizing for?**

## 5 Core Lessons

### 1. Simulation Awareness
Most actors are trapped in layered simulations (KPIs, narratives, status games) whose rules feel like reality.
**Implement:** Map incentive gradients, info asymmetries, feedback loops. Ask: "What simulation are you optimizing inside?"

### 2. Epistemic Breakout (Red Pill)
Seeing the system is costly. Truth looks worse than illusion short-term.
**Implement:** Build capacity for uncomfortable data. Create safe containers. Reward those surfacing contradictions.

### 3. Anomaly as Feature (Distributed + Resilient)
Systems need anomalies to evolve. Lone heroes get neutralized — sustainable change needs a crew. Suppressing all deviation creates brittleness.
**Implement:** Institutionalize red teams, internal startups, devil's advocates. Design for modularity and graceful degradation. Ask: "What happens when this component fails?"

### 4. Probabilistic Navigation (Oracle)
No one predicts accurately. Oracle reads probability distributions.
**Implement:** Replace forecasts with 2-4 bounded scenarios. Define signposts. When signpost fires, trigger pre-planned adjustments.

### 5. Rejection of False Binaries (Choice Architecture + Purpose)
Power presents rigged dilemmas. Real move is outside the offered frame. Purpose-driven systems absorb shocks that incentive-optimized systems cannot.
**Implement:** Pause at every binary — ask who constructed it and what outcome they prefer. Surface the third path. Anchor strategy in purpose deeper than optimization.

## Circuit Breaker

Prevents runaway analysis loops:

| State | Behavior | Trigger |
|-------|----------|---------|
| **CLOSED** | Full analysis | Normal operation |
| **OPEN** | Truncate to verdict-only | >800 tokens OR >3 recursive loops OR redundancy (same insight 3+ ways) OR user cognitive overload OR infinite regress |
| **HALF_OPEN** | One more deep analysis | After 5 min cooldown |

## Activation Protocol

1. Restate situation in Matrix language
2. Identify 2-3 most diagnostic lessons
3. Apply RRSS + KafCa to each
4. Run circuit breaker check
5. Deliver verdict-first, then analysis
